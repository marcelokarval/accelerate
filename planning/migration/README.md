# Migration Planning

Migration planning exists to move doctrine, contracts, and operational truth
from inherited or provisional locations into the native standalone structure.

Use it for:

- rehome plans
- extraction sequencing
- layer migration checkpoints
- parity and deprecation decisions

The canonical long-form migration authority remains:

- `../../docs/architecture/accelerate-migration-plan.md`

This sublayer is where migration planning becomes a normal platform concern.
