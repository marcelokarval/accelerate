# Onboarding Planning

This sublayer holds the planning bridge from onboarding discovery into actual
bootstrap execution.

Its role is to ensure that onboarding does not stop at recommendation.

The expected handoff is:

1. onboarding asks and discovers
2. onboarding recommends
3. onboarding emits an executive plan
4. later execution consumes that plan

The current canonical bootstrap artifact is:

- `../executive/bootstrap-plan-template.md`
