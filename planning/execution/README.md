# Execution Planning

Execution planning owns implementation-ready decomposition after product and
architecture truth are sufficient.

Use this sublayer after story framing, PRD-lite, SDD, or executive planning has
made the next implementation slice honest.

## Native Artifacts

- `task-breakdown-template.md`

## Rule

Execution planning must not invent product truth or architecture truth.

If actor, value, acceptance, scope, ownership, or technical design is still
unclear, return to product planning or architecture planning before creating
tasks.

## Handoff

Task breakdown should produce dependency-aware slices that can become:

- child issues
- bounded execution batches
- implementation handoff packets
- QA/proof lanes
- follow-up work
