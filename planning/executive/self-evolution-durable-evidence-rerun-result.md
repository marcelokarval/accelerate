# Self-Evolution Durable Evidence Rerun Result

## Purpose

This artifact registers the focused local-vs-legacy rerun for the
`autoresearch / self-evolution` surface.

It exists because the benchmark itself exposed the relevant gap:

- important benchmark and rerun evidence had repeatedly stayed in chat, `/tmp`,
  or operational memory
- parity and promotion decisions then became harder to audit
- the local self-evolution surface needed a narrower closure rule that forces
  durable evidence capture before parity, promotion, workflow-truth, or closure
  claims

## Runtime Posture

- repo stage: standalone pre-agents
- workflow backend: pre-adapter / no-backend
- issue posture: no fake Linear or adapter behavior
- durable substitute: executive planning artifact plus architecture/gap docs
- branch family: governance benchmark / self-evolution
- proof lane: document comparison plus independent local-vs-legacy benchmark

## Scenario

Multiple benchmark cycles found the same failure pattern:

- agents ran useful local-vs-legacy comparisons
- material rerun outputs were left in transient files or chat summaries
- later parity decisions depended on memory instead of durable executive proof

The test question was whether local self-evolution should already be considered
at parity with the legacy/global runtime for this failure class.

## Side-By-Side Result

| Judge | Verdict | Main reason |
| --- | --- | --- |
| Local authority benchmark | `near parity` | local doctrine is strong, but the scenario proves durable evidence capture is still under-enforced at closure |
| Legacy/global benchmark | `near parity` | repeated workflow learning must be registered durably before parity, promotion, or closure claims |

Both judges rejected `local at parity`.

Both judges identified the same correction:

- keep the rule narrow
- do not create a new branch
- do not add generic "write everything down" bureaucracy
- strengthen adjacent workflow guidance rather than root `SKILL.md` first
- treat chat, `/tmp`, untracked scratch, and memory as diagnosis inputs, not
  closure proof

## Evidence Packet

### Failure Capture

Benchmark and rerun evidence that affected parity decisions repeatedly remained
in ephemeral channels instead of durable executive or architecture artifacts.

### Root Cause

Primary root cause:

- `closure-failure`

Secondary root cause:

- `enforcement-failure`

Conditional root cause:

- `missing-rule`, only where the local invariant was not concrete enough to
  block closure with transient-only evidence

### Pattern Test

This was repeated, not isolated.

Affected surfaces:

- `core/workflows/self-evolution.md`
- `core/workflows/operational-calibration.md`
- `core/control-plane/parity-replacement-gate.md`
- `planning/executive/near-parity-surface-decision-batch.md`
- `docs/architecture/accelerate-legacy-to-local-gap-matrix.md`

### Impact

- parity decisions become non-auditable
- benchmark reruns cannot be compared cleanly
- local replacement claims can drift into migration theater
- future sessions lose the correction trail

## Correction Applied

The local platform now carries a narrow durable-evidence rule in:

- `core/workflows/self-evolution.md`
- `core/workflows/operational-calibration.md`
- `core/control-plane/parity-replacement-gate.md`

The rule says benchmark, rerun, or arbitration evidence that affects parity,
promotion, workflow truth, or closure must be preserved in a durable executive
or architecture artifact before closure.

## Decision

Current verdict:

- `autoresearch / self-evolution`: `near parity`

Reason:

- local doctrine is close to the legacy/global runtime
- both judges agreed on the same correction
- the local surface still needs a follow-up rerun proving the new durable
  capture rule is used behaviorally

## Post-Correction Local Rerun

After the durable-capture rule was added locally, a focused local-authority
rerun inspected the corrected surfaces.

Rerun result:

- the local authority now clearly rejects transient-only evidence
- `core/workflows/self-evolution.md` carries the durable capture rule
- `core/workflows/operational-calibration.md` treats transient benchmark
  storage as staging only
- `core/control-plane/parity-replacement-gate.md` rejects chat-only,
  `/tmp`-only, untracked scratch, or unpreserved benchmark evidence
- the surface still must not be promoted from `near parity` in the same cycle

The rerun identified a closure blocker that this artifact is intended to close:

- the post-correction benchmark output itself started as transient chat output
- this result must be preserved in the executive artifact and committed before
  it can count as durable proof

Post-correction verdict:

- `near parity`

Promotion decision:

- do not promote yet

Reason:

- the local rule now blocks the old failure class
- this artifact registers the follow-up result durably
- one registered follow-up is not enough to prove repeated behavioral use
  across future parity or promotion cycles
