# Executive Queue: Near-Parity Promotion

## Purpose

This artifact converts the current `near parity` batch into an execution queue.

It exists to answer:

- which surfaces should be promoted first
- which surfaces are mostly blocked by real reruns
- which surfaces can realistically move to `local at parity` earliest

This is not a new gap matrix.

It is the operational queue that follows the parity-decision batch.

## Promotion Rule

Promote the surfaces that are:

1. most likely to show behavioral evidence quickly
2. most likely to unblock other surfaces indirectly
3. least dependent on future-agent assumptions

Do not prioritize by document beauty or by which file is easiest to edit.
