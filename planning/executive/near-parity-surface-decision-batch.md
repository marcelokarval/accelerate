# Batch Parity Decisions: Remaining Near-Parity Surfaces

## Purpose

This artifact applies the `parity-replacement-gate` across the remaining
surfaces that are still classified as `near parity` in the current gap matrix.

It exists to stop the gate from being a one-off mechanism used only for:

- one `local ahead` surface
- and one isolated `near parity` surface

The local platform now has enough native coverage that the remaining question
is mostly not:

- “does a native surface exist?”

It is:

- “what exactly still blocks this surface from moving to `local at parity` or
  `local ahead`?”

## Scope Of This Batch

This batch covers the remaining `near parity` surfaces after the already-issued
dedicated packets for:

- `persisted-modeling review`
- `workflow catalog`

Surfaces covered here:

1. `runtime packet schema`
2. `current enforcement surfaces`
3. `issue stack`
4. `QA / proof stack`
5. `subagent model`
6. `branch enforcement matrix`
7. `operational calibration`
8. `autoresearch / self-evolution`
9. `maturity control`
10. `workflow-change approval gate`
11. `parity / replacement gate`
12. `product-critical surface discipline`
13. `premium interface discipline`

## Shared Reading Rule

For every surface below:

- native local method exists
- legacy is no longer the primary written authority
- the remaining blocker is not missing doctrine alone
- the remaining blocker is repeated operational proof, runtime coercion, or
  cross-surface repetition

So the default verdict across this batch is conservative:

- `near parity`

That is deliberate, not timid.

The gate is being used to prevent premature replacement claims now that most of
the obvious document-harvest work has already landed.

## Batch Conclusion
