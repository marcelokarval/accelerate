# Runtime-Sensitive Surface Loop Plan

## Governing Scenario

Run two separated benchmark/correction sprints on runtime-sensitive surfaces
that can still expose residual legacy advantage even after the recent QA/proof
stack promotion:

1. `auth recovery`
2. `upload/import`

Each sprint should:

1. run the local/legacy comparison at `medium` and `high`
2. identify any real local deficit
3. patch the local doctrine only if the deficit is real
4. rerun the affected side
5. issue or update the parity decision
