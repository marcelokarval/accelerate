# `.accelerate/` Project-Local Workspace Executive Plans

This index organizes the executive planning variants for the project-local
`.accelerate/` workspace.

The three plans are intentionally progressive rather than mutually exclusive.

They represent increasing levels of persisted local state:

1. [V1 Minimum Extreme](./accelerate-project-local-workspace-executive-plan-v1-minimum-extreme.md)
2. [V2 Small And Useful](./accelerate-project-local-workspace-executive-plan-v2-small-and-useful.md)
3. [V3 Broad Local Control Surface](./accelerate-project-local-workspace-executive-plan-v3-broad-local-control-surface.md)

## Reading Guidance

Use these plans as decision material, not as simultaneous implementation scope.

- `V1` is the smallest possible persistent local installation.
- `V2` is the strongest default recommendation for the first real rollout.
- `V3` is the widened target once local workflow, runtime, and memory surfaces
  become mature enough to justify more structure.

## Current Recommendation

At the current `standalone pre-agents` stage, the recommended default remains:

- `V2 Small And Useful`

`V1` is useful as a floor model.

`V3` is useful as a target-state planning model.
