# Ship Readiness Gate

Ship readiness means the branch is ready to create or update a reviewable change
proposal. It does not mean merge or deploy.

## Required Inputs

- active work item
- current plan and task ledger
- no unresolved blocker review findings
- implementation proof
- required QA and browser proof
- regression proof or explicit not-applicable rationale
- review readiness dashboard marked ready
- closure packet draft

## Blocking Conditions

- stale review
- stale QA/browser proof
- unresolved blocker finding
- missing task proof
- missing decision for one-way-door action
- provider adapter claiming live behavior without validation
