# Design-System Workflow Pipeline

Accelerate adopts the useful shape of rendered design review and approval flows
while preserving repo-local design-system authority.

## Required Flow

1. observe rendered truth
2. extract or verify design-system contract
3. compare against local design-system corpus and benchmark influence map
4. record feedback
5. record approved direction
6. implement through token, primitive, composite, shell, or page owner surfaces
7. verify screenshots at required breakpoints
8. update proof packet and readiness dashboard

## Authority

Use Accelerate's local design-system skills and corpus. External design examples
are references, not implementation authority.
