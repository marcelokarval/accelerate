# Manager Lane Model

## Purpose

This document defines the minimum native lane model required for the standalone
pre-agents phase.

## Manager Lane Pool

The root reasons through these lanes:

- executive routing
- technical
- lifecycle
- design-contract
- proof
- trust

## Ownership

These lanes are root-owned managerial functions.

They decide, route, and block. They do not primarily mutate code.

## Core Rule

Do not let a bounded implementer or reviewer silently become a manager lane.

Bounded execution returns into lane governance or back to the root.
