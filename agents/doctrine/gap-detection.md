# Agent Gap Detection

Use this module when the root must judge whether the current bounded-family
pool is insufficient.

## Core Rule

A gap is a repeated missing bounded specialty, not a one-off discomfort.

The root should call a gap only when the current pool cannot cover a stable
class of work without recurring distortion.

## Strong Gap Signals

Treat the pool as insufficient when one or more of these patterns repeat:

### 1. Repeated awkward composition

The same class of work keeps needing three-family composition to behave
honestly.

### 2. Repeated root compensation

The root keeps supplying the same missing specialty manually instead of merely
classifying and integrating.

### 3. Honest fit never exists

The dominant surface is stable and recurring, but no family can own it without:

- boundary stretch
- repeated partial fits
- write-scope confusion
- poor completion contracts

### 4. Stable branch has no family coverage

A recurring branch in the workflow remains real, but the pool has no clean
owner for it.

### 5. Misrouting keeps happening

The same kind of task repeatedly starts under different families and still
needs correction after execution starts.

## Weak Signals

Do not call a gap merely because:

- a donor repo has a nice role or TOML
- one issue felt awkward once
- a role title would look elegant
- a technology word appears frequently in prompts

## Gap Classes

### Surface gap

A recurring dominant surface has no honest owner.

### Proof-lane gap

Implementation is covered, but proof or audit keeps lacking a stable owner.

### Integration gap

The missing role is cross-boundary integration, not implementation or review in
isolation.

## Current Candidate Gaps

These remain plausible future candidates, not approved families:

- async workflow stewardship
- provider-boundary auditing
- migration stewardship

## Response To A Real Gap

When a gap is confirmed:

1. keep the current run honest
2. record the missing bounded specialty
3. feed the gap into agent-factory planning
4. evaluate recurring surfaces, risks, write scope, and skill envelope

Suggestion is not promotion.
