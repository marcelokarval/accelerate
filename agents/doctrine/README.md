# Agent Doctrine

Reserved for the standalone rehoming of agent ontology, capability, governance,
and bounded execution doctrine.

In the current `standalone pre-agents` phase, this sublayer should be read as:

- the native landing zone for agent doctrine
- the place where imported `references/codex-agents/` material is being
  re-expressed
- architectural truth about bounded agents, not proof of a live agent catalog

The doctrine sublayer now makes the first native rehome explicit through:

- `ontology.md`
- `capability-matrix.md`
- `selection-policy.md`
- `gap-detection.md`
- `authority-boundary.md`
- `pooling-model.md`

This sublayer should make explicit:

- what a bounded agent is allowed to own
- what always remains root-owned
- how family fit is judged
- how gaps are detected without forcing promotion

Operational reading order:

1. `ontology.md`
2. `capability-matrix.md`
3. `selection-policy.md`
4. `gap-detection.md`
5. `authority-boundary.md`
6. `pooling-model.md`
