# Workflow Skills

Workflow skills convert issue or plan state into executable, reviewable work.

Current skills:

| Skill | Role |
| --- | --- |
| `planning-with-files` | Persistent file-backed planning for multi-step execution. |
| `linear-implementation-planner` | Linear-specific issue-to-execution sequencing when Linear is the active backend. |
| `linear-pm` | Linear issue creation, enrichment, metadata, lifecycle, and closure hygiene. |
| `linear-progress-reporter` | Linear progress, blocker, milestone, and stakeholder-status reporting. |
| `linear-workflow-orchestrator` | Multi-issue dependency sequencing and critical-path planning. |
| `github-auth` | GitHub authentication and access-path detection across MCP, gh, git, and REST. |
| `github-issues` | GitHub issue discovery, creation, triage, and linking. |
| `github-pr-workflow` | GitHub branch, PR, CI, update, and merge workflow. |
| `github-code-review` | GitHub PR/diff review and review-comment workflow. |
| `github-repo-management` | GitHub repository setup, clone/fork, remote wiring, and identity verification. |
| `executing-plans` | Execute written plans in bounded batches with checkpoints. |
| `subagent-governance` | Multi-agent scope, delegation, nested-spawn, review aggregation, and master revalidation governance. |
| `parallel-agents` | Safe parallelization of independent reads, analyses, validations, and bounded implementation slices. |
