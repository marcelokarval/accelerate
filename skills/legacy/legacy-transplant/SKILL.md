# legacy-transplant

Use this skill when legacy is reference material for new work.

## Core Rule

Legacy is mandatory consultation material and non-canonical implementation
shape.

## Porting Rule

Preserve business truth where it still matters. Rebuild the implementation to
fit the new stack.

## Watch For

- inflated files
- collapsed boundaries
- outdated transport assumptions
- direct dependency carry-over from legacy
- copy-paste imports or naming that bypass current architecture

## Relationship To Other Skills

- use `legacy-first-protocol` to force initial consultation
- use the active stack constitution/profile to judge whether the adapted result
  fits the new stack

## Review Questions

- Was legacy consulted first?
- Was behavior preserved intentionally?
- Was old architecture left behind where needed?
