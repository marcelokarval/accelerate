# GitHub Adapter

## Purpose

This document defines the intended peer workflow adapter for GitHub.

## Status

- status: planned peer adapter
- phase: pre-agents

## Intended Contract

The GitHub adapter should mirror the same architectural semantics as Linear:

- issue bootstrap
- lifecycle truth
- hierarchy or explicit substitute model
- AI review reporting
- closure traceability

## Design Rule

GitHub is not a fallback flavor of Linear.

It is a peer adapter that must satisfy the same control-plane contract through
its own backend capabilities.
